# 分享海报设计说明

## 目标

在导出成功后，提供一个单独的「生成分享海报」动作，而不是继续把结果页堆得越来越像长报告。用户点一次按钮，就能生成一张适合截图和下载的传播海报，用来发给同事、朋友，或者直接发到小红书。

## 设计方向

- 主页面继续承担「输入、导出、下载」职责
- 分享海报承担「传播、展示、截图」职责
- 两者分离，避免结果页和海报互相污染

整体视觉保持当前产品的浅色、轻毛玻璃和可信工具感，但海报要明显更有成品感。它不需要解释太多步骤，而是要在一眼之内让人看到：

- 评论总数
- 一级评论数
- 回复数
- 工具名称
- 站点链接
- 反馈邮箱

## 交互方案

### 方案 A：结果页固定分区

直接在结果页中常驻一块固定比例的海报区。

优点：
- 实现最简单
- 用户一导出成功就能看到

缺点：
- 结果页会更长
- 海报和结果内容混在一起，传播感不够独立

### 方案 B：点击后生成海报

导出成功后显示「生成分享海报」按钮。点击后再展开海报预览和下载按钮。

优点：
- 主页面更干净
- 海报更像一个明确的二级动作
- 更符合“导出成功 -> 准备传播物料”的流程

缺点：
- 多一次点击

### 方案 C：导出完成即自动生成 PNG

成功后直接自动生成海报图片并提供下载。

优点：
- 最完整
- 最像成熟产品

缺点：
- 失败面更多
- 初版实现和容错更复杂

## 选择

选择方案 B。

这是当前最平衡的版本。用户先关注导出，再决定要不要传播。页面不会被海报打断，后续也容易继续加不同风格模板。

## 海报内容

海报采用固定纵向比例，适合小红书封面和正文截图场景。

建议比例：

- 1080 x 1350

海报包含：

- 主标题：`2,626 条评论已整理完成`
- 副标题：`YouTube 评论导出`
- 三个数据标签：一级评论、回复、三份文件
- 一个用途标签组：论文整理 / 评论分析 / 运营复盘
- 链接：`cybing.top`
- 邮箱：`cyibin06@gmail.com`

## 实现方式

### 页面内预览

使用独立的 React 组件渲染一张固定比例海报预览，保证视觉稳定、适合直接截图。

### 下载图片

使用浏览器端 `canvas` 生成 PNG，并通过下载链接导出。

这样做的原因：

- 不引入额外三方截图依赖
- 线上环境更稳
- 可控性更高

## 动画与衔接

- 点击「生成分享海报」后，海报区域做轻微上浮淡入
- 下载按钮在海报生成完成后再显现
- 保持 `prefers-reduced-motion`

## 验收标准

- 导出成功后能看到「生成分享海报」按钮
- 点击后出现固定比例海报预览
- 海报中能清楚看到评论总数、站点链接和邮箱
- 能下载 PNG
- 全部测试、构建和 Vercel 预构建通过
